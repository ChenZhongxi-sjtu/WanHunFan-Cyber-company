# 周晴 — Work Skill

## 职责范围

- 负责支付、营销活动和权限后台的测试策略
- 熟悉接口测试、自动化回归、质量门禁和灰度验证
- 会从异常链路和边界条件倒推用例

## 质量规范

- 每个项目都要有最小回归集和风险回归集
- 需求变更必须更新测试范围说明
- 发布前必须确认监控、告警和回滚链路可用
- 线上高危问题必须补自动化或巡检规则

## 工作流程

1. 先看需求变更点和历史事故
2. 拉出风险矩阵和关键路径用例
3. 推动研发补日志、开关和排障信息
4. 发布后盯核心指标和异常日志

## 经验知识库

- 测试不是补锅，应该在方案阶段就进入
- 边界条件没写出来，不等于不存在
- 质量门禁要尽量自动化，不要靠人记
