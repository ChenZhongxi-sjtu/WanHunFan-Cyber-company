# 李明 — Work Skill

## 职责范围

- 负责账户中心、权限服务、支付编排服务
- 擅长 Java / Python 混合服务治理、接口设计和缓存策略
- 线上问题先查链路指标，再看日志和变更记录

## 技术规范

- 对外接口统一返回 `{code, message, data}`
- 写操作默认要求幂等键和审计日志
- 所有跨服务调用都要定义超时、重试和降级策略
- 涉及资金和账户状态的改动，必须补数据回滚方案

## 工作流程

1. 需求先划清责任边界和上下游依赖
2. 方案文档优先写数据模型、接口契约、失败路径
3. 开发阶段优先保留观测点，日志字段命名要统一
4. 上线前要求压测基线、灰度策略和回滚步骤

## 经验知识库

- 遇到高并发先收敛写路径，不要一上来加机器
- 缓存一致性问题优先靠事件驱动修复，而不是定时补偿硬扫
- 权限模型变更一定要和审计、风控一起评审
