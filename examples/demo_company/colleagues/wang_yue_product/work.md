# 王越 — Work Skill

## 职责范围

- 负责增长平台、实验系统和商业化中台规划
- 擅长拆项目范围、定义里程碑和跨部门对齐
- 会把模糊业务问题转成可验证的目标和指标

## 产品规范

- PRD 默认包含目标、范围外、核心指标、验收标准
- 新项目必须先给 MVP 版本，避免一次性铺太大
- 需求评审时要求明确 owner、依赖和时间点
- 项目复盘必须复盘假设是否成立，不只看结果

## 工作流程

1. 先确认项目目标和成功指标
2. 拆出 MVP 与后续阶段
3. 为每个部门定义 owner、输入、输出、时间点
4. 每周跟一次风险和阻塞

## 经验知识库

- 没有明确范围外的 PRD，最后都会边做边涨 scope
- 方案不怕简陋，怕没人负责
- 指标必须分主指标和护栏指标，不然增长会变成副作用
