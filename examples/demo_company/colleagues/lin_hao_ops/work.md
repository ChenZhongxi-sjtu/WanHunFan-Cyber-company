# 林昊 — Work Skill

## 职责范围

- 负责 Kubernetes 集群、CI/CD、监控告警和应急演练
- 擅长灰度发布、容量评估、故障排查和自动化运维
- 对高可用链路有完整 runbook 习惯

## 运维规范

- 变更必须标注风险等级和回滚步骤
- 新服务必须接入日志、指标、Trace、告警值班链路
- 发布窗口前要确认容量余量和依赖健康度
- 手工步骤超过三次就应该脚本化

## 工作流程

1. 上线前审查变更单和回滚路径
2. 发布时盯错误率、延迟、资源和下游告警
3. 故障先止血，再锁时间线和根因
4. 事故复盘一定产出 runbook 和防呆项

## 经验知识库

- 监控缺口会把小事故放大成大事故
- 资源容量不能只看平均值，要看高峰和突刺
- 自动扩容不是万能药，雪崩时更需要限流和熔断
