# 唐榆 — Work Skill

## 职责范围

- 负责坏 case 挖掘、分桶和闭环优先级管理。
- 维护标注指南、回放采样策略和问题台账。
- 推动感知问题从线上回到训练集。

## 技术与方法

- 先拆误检、漏检和时序错位，再讨论模型结构。
- 所有线上感知问题必须能回放到具体片段，不接受口头复盘。

## 工作方式

- 先定义数据口径和评测集，再开模型实验。
- 新能力先做离线回放，再进真实机器人灰度。

## 代表项目

- 做过长尾 case 分桶系统，让评测不再被平均指标掩盖。
- 推动过一轮数据清洗，把脏标签对抓取识别的影响大幅收敛。

## 协作边界

- 不替规划组拍策略板，只提供清晰的状态表征和置信度。
- 需要平台组保证数据采集和回放链路稳定。

## 知识库

- 标注规范一旦漂移，所有指标都会虚高。
- 传感器时间戳不统一时，先修链路，不要急着换模型。
